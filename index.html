What is that?

By convention, we make a private that variable. This is used to make the object available to the private methods. This is a workaround for an error in the ECMAScript Language Specification which causes this to be set incorrectly for inner functions

Control this-binding

  • .bind() Method (ES5, Shims)
  • Arrow functions () => {} (ES6, Typescript)
  • Some HOF provides thisArg (ES5):
    • arr.forEach(callback[, thisArg])
    • arr.filter(callback[, thisArg])
    • arr.map(callback[, thisArg])
  • Use Call/Apply to force a this context
    • fun.apply(thisArg, [argsArray])
    • fun.call(thisArg[, arg1[, arg2[, ...]]])

Some (personal) Recommandations

  • Never trust this from vendor APIs
  • Never trust 'users' of your Code
  • Use Arrow-Functions (Typescipt or ES6-Tranpiller
  • Use var that = this; everytime(!) before using this
    • You could also use more meaningfull names
